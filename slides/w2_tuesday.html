üóÇÔ∏è The Four Main Data Structures

Python gives us four powerful (built-in) tools to organize business data:


Structure Best For Business Example Key Feature
List Ordered sequences Daily sales figures, customer queue Order matters

The Goal: Match your data‚Äôs behavior to the right structure

Which One Matters


Choosing the right one makes your code:

  • Easier to write
  • Faster to run
  • Simpler to understand


For example‚Ä¶

Example 1: Dictionary


üîé Goal: You need to look up a product price by name.

prices = {"apple": 0.99, "banana": 0.59}


  • This is a dictionary (dict)
  • It uses key‚Äìvalue pairs surrounded by {}
  • Keys are "apple", "banana"; values are prices

Important

‚úÖ A dictionary allows fast key-based lookup.

üìå Use when you need to map one thing (e.g. name) to another (e.g. price).

Example 2: Tuple


üìç Goal: You‚Äôre storing a GPS coordinate that should never change.

location = (39.76, -84.19)


  • This is a tuple
  • Ordered, but immutable (cannot be changed)
  • Surrounded by () and contains values in sequence

Important

‚úÖ A tuple protects data from being changed accidentally.

üìå Use when data is fixed and position matters.

Example 3: List


üéß Goal: You‚Äôre storing songs in a playlist, in order.

playlist = ["Intro", "Track 1", "Track 2"]


  • This is a list
  • Ordered and mutable (can add/remove/change items)
  • Uses [] brackets for creation

Important

‚úÖ A list lets you store sequences of items and modify them easily.

üìå Use when order matters and you‚Äôll be changing the contents.

Example 4: Set


üìß Goal: You‚Äôre collecting unique email addresses for a newsletter, no duplicates allowed.

subscribers = {"john@email.com", "sarah@email.com", "mike@email.com"}


  • This is a set
  • Unordered but mutable (can add/remove items)
  • Uses {} brackets but NO key-value pairs (just values)
  • Automatically removes duplicates

Important

‚úÖ A set ensures all items are unique and provides fast membership testing.

üìå Use when you need to eliminate duplicates or check ‚Äúis this item in the collection?‚Äù

üéØ Business Decision Framework

When choosing a data structure, ask yourself:

  1. Do I need to look things up by name/ID? ‚Üí Dictionary
  2. Is the order of items important? ‚Üí List or Tuple
  3. Will the data change over time? ‚Üí List (if yes), Tuple (if no)
  4. Do I need only unique values? ‚Üí Set


Important

Key Insight: The right structure makes your analysis faster, clearer, and more maintainable!

If you needed to calculate the correlation between two variables (x and y)

üõ†Ô∏è Build It Yourself

mean_x = sum(x) / len(x)
mean_y = sum(y) / len(y)

numerator = sum((a - mean_x)*(b - mean_y) for a, b in zip(x, y))
denominator = (
    sum((a - mean_x)**2 for a in x) *
    sum((b - mean_y)**2 for b in y)
) ** 0.5

correlation = numerator / denominator
print(f"correlation = {correlation}")

Python Ecosystem

Tip

Python‚Äôs true power comes from its ecosystem of packages, libraries, and modules that extend its core functionality

Vocabulary

There are terminology differences between‚Ä¶

  • Module: A single .py file that contains Python code‚Äîfunctions, variables, classes‚Äîthat you can reuse. For example, the math module lets you do mathematical calculations.
  • Library: A collection of related modules bundled together. For example, pandas is a library that includes several modules for data manipulation.
  • Package: A directory containing one or more modules or libraries, with an init.py file that tells Python it‚Äôs a package. You can think of a package as the container that holds libraries and modules.

But I‚Äôm not hung up in you knowing these differences at this time. More importantly, I want you to understand the following‚Ä¶
